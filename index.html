Prework Study Guide

HTML

  • The head element contains information about the webpage
  • The body element represents the visiable content shown to the user
  • Elements that are nested inside other elements are called child elements
  • name of the element describes its content and the role it plays on a webpage
  • When using headings, it is important that we use heading tags in order
  • Declarations are not case-sensitive; HTML elements are case-sensitive
  • HTML attributes are always defined in the beginning element, never the closing element
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage to users
  • style element defines the CSS styles associate with a website
  • base element defines the base URL for a webpage
  • link element connects an external resource to document
  • meta element defines metadata such as the character set, description, keywords, author and viewpoint
  • the script element embeds client-side scripts in the document

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS: applies to a specific tag directly within html
  • Internal CSS: applies to entire page
  • External CSS: keeps designs in a separate file - preferred method
  • Separation of Concerns: separating codes into files based on use
  • Elements of CSS Rule: Property and Value must be included in declarations block
  • Color assignments can be semantic (Red,Blue) or Hexidecimal Code (hex#)

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches into it
  • git add -A: adds branch to track
  • git commit -m descriptive message: commits staged content as a new commit snapshot
  • git push: transmits local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch
